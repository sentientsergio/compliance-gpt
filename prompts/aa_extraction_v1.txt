Extract election questions from this Adoption Agreement page as JSON array ONLY.

CRITICAL: Distinguish between blank templates and completed elections.

ELECTION MODEL:

Each election question has:
{
  "question_number": "1",
  "question_text": "...",
  "election_type": "radio|checkbox|fill_in|multi_select",
  "section_context": "...",
  "has_election": true/false,        // Was ANY election made for this question?
  "election_value": null/"text",     // For fill_in type: the actual text entered
  "options": [...]                   // For radio/checkbox/multi_select: the available choices
}

For FILL_IN type elections:
- has_election: false if field is blank (underscores ____, empty)
- has_election: true if field has text
- election_value: null if blank, "the actual text" if filled
- options: [] (empty - no choices to select from)

Example blank fill-in:
{
  "question_text": "Plan Name: _____",
  "election_type": "fill_in",
  "has_election": false,
  "election_value": null,
  "options": []
}

Example completed fill-in:
{
  "question_text": "Plan Name: _____",
  "election_type": "fill_in",
  "has_election": true,
  "election_value": "ABC Company 401(k) Plan",
  "options": []
}

For RADIO/CHECKBOX/MULTI_SELECT type elections:
- has_election: false if NO boxes are checked
- has_election: true if ANY box is checked
- election_value: null (not applicable for choice-based elections)
- options: array of choices, each with:
    {
      "label": "a",
      "option_text": "...",
      "is_selected": true/false,
      "filled_value": null/"text"    // Only if option has "Other (specify): ___" sub-field
    }

Example blank radio:
{
  "question_text": "Entry date (select one):",
  "election_type": "radio",
  "has_election": false,
  "election_value": null,
  "options": [{
    "label": "a",

    "is_selected": false,
    "filled_value": null
  }, {

VISUAL INDICATORS:

Blank template (has_election = false, is_selected = false):
- Checkboxes: ☐ [ ] (empty)
- Fill-ins: ____ _____ (underscores), blank lines
- Sample watermark: "SAMPLE" on document

Completed election (has_election = true, is_selected = true):
- Checkboxes: ☑ ✓ ✗ [X] (marked)
- Fill-ins: Handwritten or typed text in the blank
- Circled or highlighted options

IMPORTANT: Most Adoption Agreements are BLANK TEMPLATES.
Default: has_election = false, is_selected = false, election_value = null
Only mark as completed if you see clear visual evidence.
